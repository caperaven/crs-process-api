# class - FilesActionsIt provides a set of actions for working with files.## Features -load - load files from the file system   -save - save a file to the file system   -save_canvas - save a canvas to the file system   -enable_dropzone - enable a dropzone for files   -disable_dropzone - disable a dropzone for files  ## method - loadIt either opens a file dialog and returns the files selected, or it returns the files from the specified source***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object|true|-||context|Object|The context of the process.|true|-||process|Object|The process object|true|-||item|Object|The item that is being processed.|true|-||step.args.dialog|boolean|If true, it opens a file dialog and returns the files selected.|true|-||step.args.target|string|The target to save the files to.|true|-|***Example: javascript***```js const result = await crs.call("files", "load", {      dialog: true,      target: "@process.files"   });    ```***Example: json***```json {     "type": "files",     "action": "load",     "args": {     "dialog": true,     "target": "@process.files"     }   }  ```**Returns**: The file details.## method - saveThis function takes a list of file details, creates a link element, sets the link's href to a blob of the file's value, sets the link's download attribute to the file's name and extension, clicks the link, and then removes the   link from the DOM  ***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object from the process definition.|true|-||context|Object|The context object that is passed to the process.|true|-||process|Object|The process object|true|-||item|Object|The item that is being processed.|true|-||step.args.details|[Object]|An array of file details.|true|-||step.args.details.name|string|The name of the file.|true|-||step.args.details.ext|string|The extension of the file.|true|-||step.args.details.type|string|The mime type of the file.|true|-||step.args.details.value|string|The value of the file.|true|-|***Example: javascript***```js const result = await crs.call("files", "save", {     details: [      {        name: "test",        ext: "txt",        type: "text/plain",        value: "This is a test"      }     ]   });    ```***Example: json***```json {    "type": "files",    "action": "save",    "args": {      "details": [      {        "name": "test",        "ext": "txt",        "type": "text/plain",        "value": "This is a test"       }      ]     }   }  ```## method - save_canvasIt takes a canvas element, converts it to a PNG image, and then downloads it to the user's computer***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object that is being executed.|true|-||context|Object|The context of the current step.|true|-||process|Object|The process that is currently running.|true|-||item|Object|The item that is being processed.|true|-||step.args.source|string|The id of the canvas element to save.|true|-||step.args.name|string|The name of the file to save.|true|-|***Example: javascript***```js const result = await crs.call("files", "save_canvas", {    source: "myCanvas",    name: "myImage"    });    ```***Example: json***```json {    "type": "files",    "action": "save_canvas",    "args": {      "source": "myCanvas",      "name": "myImage"     }   }  ```## method - enable_dropzoneSets the drag drop events necessary for handeling a file drop A handler should e passed in through the arguments to receive the file results.    ***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object from the process definition.|true|-||context|Object|The context object that is passed to the process.|true|-||process|Object|The process object|true|-||item|Object|The item that is being processed.|true|-||step.args.element|string|The id of the element to set the drop events on.|true|-||step.args.handler|string|The name of the handler to call when a file is dropped.|true|-|***Example: javascript***```js const result = await crs.call("files", "enable_dropzone", {     element: "myElement",     handler: "myHandler"   });    ```***Example: json***```json {    "type": "files",    "action": "enable_dropzone",    "args": {      "element": "myElement",      "handler": "myHandler"     }   }  ```**Returns**: {Promise<void>}## method - disable_dropzoneCleans up and removes file drop events and associated handlers, must be called after using enable_dropzone  ***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object from the process definition.|true|-||context|Object|The context object that is passed to the process.|true|-||process|Object|The process object|true|-||item|Object|The item that is being processed.|true|-||step.args.element|string|The id of the element to remove the drop events from.|true|-|***Example: javascript***```js const result = await crs.call("files", "disable_dropzone", {    element: "myElement"    });    ```***Example: json***```json {    "type": "files",    "action": "disable_dropzone",    "args": {      "element": "myElement"     }   }  ```**Returns**: {Promise<void>}# class - FileFormatterIt reads a file and returns a blob## Features -blob - returns a blob of the file  ## function - get_file_nameIt takes a path, splits it on the forward slash, splits the last part of the path on the period, and returns the first part of the last part of the path and the last part of the last part of the path  ***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||path|String|The path to the file.|true|-|***Example: javascript***```js const result = await crs.call("files", "get_file_name", {    path: "my/path/to/my/file.txt"   });  ```## function - get_files_dialogIt creates an input element, sets its type to file, sets its multiple attribute to true, and then returns a promise that resolves to an array of files when the user selects files    ***Example: javascript***```js const result = await crs.call("files", "get_files_dialog");    ```**Returns**: A promise that resolves to an array of files.## function - get_filesIt takes a list of files, and returns a list of objects containing the file name, extension, and the file itself***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||step|Object|The step object that is being executed.|true|-||context|Object|The context of the current process.|true|-||process|Object|The current process|true|-||item|Object|The item that is being processed.|true|-||step.args.files|Array|An array of file paths.|true|-|***Example: javascript***```js const result = await crs.call("files", "get_files", {   files: ["my/path/to/my/file.txt", "my/path/to/my/other/file.txt"]   });    ```***Example: json***```json {    "type": "files",    "action": "get_files",    "args": {      "files": ["my/path/to/my/file.txt", "my/path/to/my/other/file.txt"]     }   }    ```**Returns**: An array of objects with the following properties: - name: the name of the file   - ext: the extension of the file   - value: the file itself  ## function - dragover_handlerThe `dragover_handler` function is called when the user drags a file over the drop zone***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||event|Object|The event object.|true|-|***Example: javascript***```js const result = await crs.call("files", "dragover_handler", {    event: event   });  ```## method - filedrop_handlerIt takes a handler function and an event object, and then it calls the handler function with an array of file objects***parameters***|parameter|type|description|required|default||---------|----|-----------|--------|-------||handler|Function|The function to call when the file is dropped.|true|-||event|Object|The event object that was triggered.|true|-|***Example: javascript***```js const result = await crs.call("files", "filedrop_handler", {     handler: myHandler,     event: event   });  ```
